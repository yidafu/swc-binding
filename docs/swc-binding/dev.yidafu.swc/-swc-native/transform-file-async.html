fun transformFileAsync(filepath: String, isModule: Boolean, options: Options, onSuccess: (TransformOutput) -> Unit, onError: (String) -> Unit)

Kotlin-friendly async transform file with lambda callbacks.

This method reads a file from disk and transforms it asynchronously in a background thread. The callbacks are invoked when the operation completes or fails. The calling thread is not blocked.

Transformation can include transpiling (e.g., TypeScript to JavaScript), applying plugins, minification, and other code modifications based on the provided options.

This method is also Java-friendly. Java code can use anonymous inner classes or lambda expressions (Java 8+) for the callbacks.

For Kotlin code, prefer using the suspend function transformFileAsync instead, which provides better integration with coroutines and structured concurrency.

Parameters

filepath

Absolute or relative path to the source file

isModule

Whether the file should be treated as a module (affects parsing behavior)

options

Transform configuration (use options DSL to create)

onSuccess

Lambda called with TransformOutput containing transformed code when successful

onError

Lambda called with error message string when transformation fails

suspend fun transformFileAsync(filepath: String, isModule: Boolean, options: Options): TransformOutput

Coroutine-based async transform file (suspend function).

This method reads a file from disk and transforms it asynchronously using coroutines. Transformation can include transpiling (e.g., TypeScript to JavaScript), applying plugins, minification, and other code modifications. It integrates seamlessly with Kotlin coroutines and structured concurrency.
